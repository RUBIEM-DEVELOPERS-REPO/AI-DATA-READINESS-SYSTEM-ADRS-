PRs means Pull Requests.
A Pull Request (PR) is when a developer finishes some code (like TK finishing /evidence/upload), then they “request” that the team reviews it before merging it into the main codebase.
In simple terms:
? Developer writes code ? pushes it to GitHub/GitLab
?? Opens a PR
?? Team reviews it
?? Fixes are made if needed
? Then it gets merged into main

Why PRs matter for your project
Because PRs prevent the team from accidentally:
* changing CDM field names (schema drift)
* breaking ICD endpoints
* skipping audit logging
* missing RBAC rules
So when Its said “Review PRs for schema drift”, Its meant:
Wills should review every pull request to ensure the code still matches the CDM + ICD exactly.
